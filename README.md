# Vue Auth App

## 1. Формулировка задачи

Цель проекта — создать многостраничное SPA-приложение на Vue 3 с авторизацией через внешний API, защищёнными маршрутами и работой с токенами.

Задача заключается в том, чтобы пользователь мог войти в систему, просмотреть список пользователей и получить доступ к приватной странице профиля, которая защищена навигационным guard и открывается только при наличии действующего токена.

Приложение реализовано в виде SPA с использованием Vue Router, Composition API и localStorage для хранения токена авторизации.

## 2. На какую аудиторию рассчитываем

Приложение рассчитано на пользователей, которым необходима авторизация для доступа к персональным данным.

Пользователь вводит логин и пароль, получает токен и попадает на приватную страницу профиля с полной информацией об аккаунте. Без авторизации доступ к странице профиля закрыт — роутер автоматически перенаправляет на страницу входа.

## 3. Что умеет приложение

Реализованы 5 страниц:

- Главная (/) — приветственная страница с описанием, иллюстрацией и кнопкой перехода на авторизацию
- Пользователи (/users) — список пользователей из внешнего API с именем, фамилией и email каждого
- Авторизация (/login) — форма с полями логин и пароль, при ошибке отображается сообщение от сервера внутри формы
- Профиль (/profile) — приватная страница с информацией об авторизованном пользователе включая фото, доступна только при наличии токена
- Регистрация (/register) — форма создания нового пользователя, после успешной отправки отображается ID нового пользователя

## 4. Как устроено приложение

Проект реализован на **Vue 3** с использованием **Composition API**, **Vue Router 4** и **Vite**.

Структура проекта:

- `router/index.js` — настройка маршрутов и navigation guard, который проверяет наличие токена в localStorage перед открытием страницы /profile
- `views/HomeView.vue` — главная страница с кнопкой перехода на авторизацию
- `views/UsersView.vue` — GET-запрос на https://dummyjson.com/users, отображение списка пользователей
- `views/LoginView.vue` — POST-запрос на https://dummyjson.com/auth/login, сохранение токена в localStorage под ключом token
- `views/ProfileView.vue` — GET-запрос на https://dummyjson.com/auth/me с заголовком Authorization: Bearer token
- `views/RegisterView.vue` — POST-запрос на https://dummyjson.com/users/add, отображение ответа сервера

Логика авторизации:

- Пользователь вводит логин и пароль на странице /login
- Отправляется POST-запрос, сервер возвращает accessToken
- Токен сохраняется в localStorage под ключом token
- Navigation guard проверяет наличие токена при переходе на /profile
- Запрос на /auth/me выполняется с заголовком Authorization: Bearer token
- При выходе токен удаляется из localStorage, пользователь перенаправляется на /login

## 5. Как запустить

Установить зависимости и запустить dev-сервер:

```
npm install
npm run dev
```

Приложение откроется по адресу http://localhost:5173

Тестовые данные для входа: логин emilys, пароль emilyspass

Полный список пользователей и их данных для входа доступен по адресу https://dummyjson.com/users
