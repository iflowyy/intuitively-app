# Проектный семинар с фреймворками JavaScript — КТ №2

## Тема задания

Разработка SPA-приложения с авторизацией на фреймворке Vue 3 с использованием маршрутизации и внешнего API.



## Формулировка задачи (Task)

Реализовать многостраничное приложение на Vue 3 с применением Vue Router. Приложение должно включать авторизацию через FAKE API (dummyjson.com), защищённый маршрут и работу с токенами через localStorage.



## Технологии

- Vue 3 (Composition API)
- Vue Router 4
- Vite
- DummyJSON API (https://dummyjson.com)
- localStorage для хранения токена



## Реализованные страницы

**1. Главная (/)home**
Страница приветствия с описанием, иллюстрацией и кнопкой перехода на страницу авторизации.

**2. Пользователи (/users)**
Получение списка пользователей через GET-запрос на `https://dummyjson.com/users`. Отображаются имя, фамилия и email каждого пользователя. Авторизация не требуется.

**3. Авторизация (/login)**
Форма с полями логин и пароль. POST-запрос на `https://dummyjson.com/auth/login`. При успешной авторизации токен сохраняется в localStorage под ключом `token`, пользователь перенаправляется на страницу профиля. При ошибке — сообщение об ошибке отображается внутри формы.

**4. Профиль (/profile) — приватная страница**
Перед входом маршрутизатор проверяет наличие токена в localStorage. Если токена нет — редирект на /login. Если есть — GET-запрос на `https://dummyjson.com/auth/me` с заголовком `Authorization: Bearer <token>`. Отображается полная информация об авторизованном пользователе включая фото.

**5. Регистрация (/register)**
Форма создания нового пользователя через POST-запрос на `https://dummyjson.com/users/add`. После успешного создания отображается ответ сервера с ID нового пользователя.



## Логика авторизации

1. Пользователь вводит логин и пароль на странице /login
2. Отправляется POST-запрос с данными в body
3. Сервер возвращает `accessToken`
4. Токен сохраняется в `localStorage` под ключом `token`
5. Навигационный guard в роутере проверяет наличие токена при попытке открыть /profile
6. Запрос на /auth/me выполняется с заголовком `Authorization: Bearer <token>`
7. При выходе токен удаляется из localStorage, пользователь перенаправляется на /login



## Ответы на вопросы по теме

**Что такое Vue Router?**
Vue Router — официальная библиотека маршрутизации для Vue.js. Позволяет создавать SPA с несколькими страницами без перезагрузки браузера, управлять историей переходов и защищать маршруты с помощью navigation guards.

**Что такое Navigation Guard?**
Navigation Guard — это функция-перехватчик в Vue Router, которая выполняется перед переходом на маршрут. Используется для проверки условий доступа. В данном проекте guard проверяет наличие токена в localStorage перед открытием страницы /profile.

**Что такое localStorage?**
localStorage — веб-хранилище браузера, которое сохраняет данные в виде строк без срока истечения. Данные сохраняются между сессиями. В проекте используется для хранения JWT-токена после авторизации.

**Что такое JWT-токен?**
JSON Web Token — стандарт передачи данных аутентификации. Состоит из трёх частей: header, payload, signature. Сервер выдаёт токен при входе, клиент прикладывает его в заголовок каждого защищённого запроса.

**Что такое Composition API?**
Подход к написанию компонентов в Vue 3 через функции `setup`, `ref`, `reactive`, `onMounted` и другие. В отличие от Options API позволяет лучше переиспользовать логику и организовывать код по функциональным блокам.
